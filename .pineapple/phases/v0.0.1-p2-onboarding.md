# P2 — `v0.0.1-p2-onboarding` — Signup + verify + onboarding

**Goal:** *Can a personal or customer user self-serve through native signup → OTP
verify → onboarding and land on the correct role home against **alpha.6** backend?*

**Status:** RUNNING (orchestrated 2026-07-11). Prerequisite: P1 app shell shipped.
Taskgraph: [`v0.0.1-p2-onboarding-taskgraph.md`](v0.0.1-p2-onboarding-taskgraph.md).

## In-scope

| Slice | Delivers | Spec |
|---|---|---|
| **core/auth** (OTP wire) | verify-email, resend-verify, reset-password/verify | [`otp-email-verification.md`](../features/otp-email-verification.md) |

## Out-of-scope

- Session/dispatch/call MVP (full P2)
- Realtime WSS
- Affiliations feature (copy-only handoff in personal wizard)
- LSP signup (leo-web)

## Backend dependency

`leo-api` **alpha.6+** OTP contract:
- `POST /auth/verify-email { email, code }`
- `POST /auth/resend-verify { email }`
- `POST /auth/reset-password/verify { email, code }` → `reset_ticket`

## E2E smoke checklist (P2-O-T-05 — manual)

Run against staging with alpha.6+ API.

### Personal funnel

- [ ] `/signup` → Personal → details → submit
- [ ] Router redirects to `/verify-email` (no imperative push)
- [ ] Enter OTP from email → session minted
- [ ] Router → `/onboarding/personal` when `onboardingRequired`
- [ ] Complete profile → `/idle` (tenant-less)
- [ ] Affiliation note visible on final step

### Customer funnel

- [ ] `/signup` → Customer → details (org + timezone) → submit
- [ ] OTP verify → `/onboarding/customer`
- [ ] Org profile + team invite → `/call`

### Edge cases

- [ ] Login with unverified email → `/verify-email?source=login`
- [ ] LSP card opens external leo-web URL
- [ ] Wrong OTP shows error; resend works (cooldown UI)

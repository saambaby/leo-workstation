# P2 delta — `v0.0.1-p2-onboarding-lsp` — In-app LSP signup

**Goal:** *Can an LSP operator complete account creation in-app — picker → details →
OTP verify → MFA enroll — and land on `/dispatch`, without leaving the app?*

**Status:** Taskgraph **approved** (Gate 1, 2026-07-11) — not yet orchestrated.
Taskgraph: [`v0.0.1-p2-onboarding-lsp-taskgraph.md`](v0.0.1-p2-onboarding-lsp-taskgraph.md).
Prerequisite: `v0.0.1-p2-onboarding` (this delta reuses its OTP-verify wire and
`auth`'s MFA-enroll screen — see that phase's taskgraph). **Scheduling relative to
that phase (separate pass after it closes vs. folded in before) is an open founder
call** — see `lsp-native-signup.md` Open Questions and `.pineapple/state.md`. This
taskgraph can be approved and orchestrated independently either way; it only touches
files the parent phase's tasks don't own (see `must_not_touch` below).

## Out-of-scope

- LSP languages/partners/pricing onboarding wizard (stays `leo-web`)
- Any new `leo-api` route or DTO change (backend union already accepts `business_type: 'lsp'`)
- Removing the `leo-web` LSP signup entry point (stays as an alternate)
- Admin CRUD / rates / billing / users routes (`INV-CLIENT-ROUTE-1` back-office half, unchanged)

## Backend dependency

None new. Reuses `leo-api` alpha.4+'s existing `POST /auth/signup` union
(`business_type: 'lsp'`), `POST /auth/verify-email`, and `POST /auth/mfa/enroll` —
all already wired by `v0.0.1-p2-onboarding` / `v0.0.1-alpha.1-auth-live`.
